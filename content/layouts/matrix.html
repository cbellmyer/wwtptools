[ What This Will Be ]

The Process Decision Matrix is a structured troubleshooting tool designed for field use. You start with an observable symptom — rising effluent TSS, dropping DO, digester foaming, polymer not holding — and the matrix walks you through a decision tree of possible causes and corrective actions, weighted by frequency and severity.

Unlike static reference tables, the matrix is interactive: branch points narrow the diagnostic path based on what you observe, what you've already ruled out, and what your process data shows. The goal is to get from "something's wrong" to "here's what to check and in what order" in under two minutes.

[ Help Build It ]

The matrix is only as good as the operational experience behind it. If you've worked through a process upset and know the diagnostic path — what you checked, what the cause turned out to be, what actually fixed it — that's exactly what this tool needs.

Open a GitHub Issue to suggest diagnostic paths, common failure modes, or zones that should be prioritized. You don't need to know how to code — describe the symptom, the causes you've seen, and the corrective actions that worked.
